ℹ️ About this Adapter

ioBroker Sigenergy Adapter
Version …
Modbus V2.5

This adapter enables integration of Sigenergy solar energy systems into ioBroker via the Modbus TCP or RS485 RTU protocol. It supports reading all plant and inverter data as defined in the Sigenergy Modbus Protocol V2.5 (released 2025-02-19).

📡 Default Modbus Addresses

🏭 Plant / Power Plant 247 (0xF7)
🔄 Plant Broadcast (no reply) 0
⚡ Hybrid Inverter 1
🔌 AC Charger (EVAC) 2
📡 Modbus TCP Port 502
🔗 RS485 Baud Rate 9600 bps

📋 Register Ranges

Read-Only Registers (Input — FC 0x03)

30000–30087 Plant running info (slave 247)
30500–31065 Hybrid inverter running info
31500–31508 DC Charger info
32000–32014 AC Charger running info

Holding Registers (FC 0x04 / 0x06 / 0x10)

40000–40044 Plant parameter setting (slave 0 or 247)
40500–40501 Inverter Start/Stop + Grid code
41000 DC Charger Start/Stop
41500–41507 Inverter EMS dispatch control
42000–42001 AC Charger control

⚠️ Notes & Tips

  • Make sure the inverter is reachable via Ethernet or RS485 before configuring this adapter.
  • For TCP connections, the inverter acts as a Modbus TCP Server on port 502.
  • For RS485, connect only one device — it will proxy all other devices in the plant.
  • Minimum poll interval is 5000 ms. Sigenergy recommends ≥ 1000 ms between requests.
  • AC Charger (EVAC) requires a connected Hybrid Inverter — access is via the inverter's Modbus interface.
  • DC Charger registers are accessible via the inverter's Modbus slave address.
